Portfolio / Power Five

Top-tier projects built for data at scale

Each project below highlights the reason it exists, the hard technical constraints, and the tool-specific solution used to make it production ready.

1) Full-Stack ML App - Streamlit + Scikit-learn

Purpose: Deliver an end-to-end prediction tool that turns model outputs into business-facing decisions.

Challenges: Feature drift in live inputs, model versioning discipline, and keeping inference latency low in a lightweight UI stack.

Solutions: Built a clean training pipeline in Scikit-learn, versioned artifacts, and deployed a Streamlit interface with validated feature schemas for stable real-time predictions.

2) Kafka KRaft Event Pipeline

Purpose: Replace batch-heavy data flow with event streaming for faster operational decisions.

Challenges: KRaft mode broker-controller configuration, partition strategy, and consumer lag during traffic spikes.

Solutions: Tuned topic partitioning, applied idempotent producers, and used offset monitoring plus retry policies to maintain throughput consistency.

3) Spark ETL on Cloud Object Storage

Purpose: Normalize large, messy datasets into analytics-ready tables for reporting and ML.

Challenges: Skewed joins, memory pressure, and long-running jobs on uneven partitions.

Solutions: Applied repartitioning, caching strategy, and optimized Spark transformations with schema-first data contracts.

4) Hadoop Data Lake Foundation

Purpose: Create low-cost storage and historical retention for high-volume records.

Challenges: Ingestion reliability, data quality checks, and maintaining query performance over growing archives.

Solutions: Built staged ingestion layers with validation checkpoints and partition-aware organization for faster downstream access.

5) Cloud Monitoring and Reliability Dashboard

Purpose: Make data platform health visible through actionable reliability signals.

Challenges: Alert noise, delayed incident detection, and unclear ownership during outages.

Solutions: Defined SLI and SLO thresholds, grouped alerts by service criticality, and documented response runbooks to accelerate resolution.
